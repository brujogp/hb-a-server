Introducción

Basados en los principios simples de REST, los endpoint de Heroclix Bible Web API regresan metadatos en JSON acerca de diferentes secciones del juego, como Reglas Completas, ATA's y Erratas directamente desde el catálogo de Heroclix Bible.

Tu aplicación

Heroclix Bible

La dirección base del Web API es https://api.heroclixbible.com. La API suministra un conjunto de endpoints con sus propios path's únicos.

Requests a HeroclixBible

La API Web de HeroclixBible está basado en REST. Los recursos de datos son accedidos mediante peticiones estándares HTTPS en el formato UTF-8 al endpoint de la API. Los siguientes verbos HTTP están permitidos:

MÉTODOS ACCIONES
GET Devuelve recursos

Versionamiento

Esta vez, decidimos comenzar por el principío, y ya que actualmenten esta API está en la versión 1.0.0, todas las peticiones tendrán que llevar en la URL de la petición, la versión actual:

EJEMPLO
https://api.heroclixbible.com/v1/

Estructura de recursos

La división de recursos tiene 3 niveles los cuales puedes consultar directamente mediante algún endpoint especializado.
Los niveles son:

- Secciones

- Subsecciones

- Nodos

Lista de endpoints

Mediante la dirección base, puedes construir llamar y solicitudes personalizadas a diferentes recursos mediante la siguiente lista de endpoints con los respectivos verbos HTTP:

NIVEL ENDPOINT's PARÁMETROS ACCIONES
Sección /v1/sections q Solicita una lista de secciones que coincidan con un String de consulta mediante el parámetro q
Sección /v1/sections/ - Retorna toda la lista de secciones disponibles a partir del lenguaje *
Sección /v1/sections/{acronym} - Retorna una sección a partir de su acrónimo y del lenguaje especificado *
Sub-sección /v1/subsections q Retorna una lista de subsecciones que coincidan con un String de consulta mediante el parámetro q
Sub-sección /v1/subsections/ - Retorna una lista de sub- secciones disponibles a partir del lenguaje especificado *
Sub-sección /v1/subsections/{id}/parent - Retorna el padre (la sección) de la sub-sección mediante la propiedad ID
Poderes y Habilidades /v1/paa q Solicita una lista de Poderes y Habilidades que coincidan con un String de consulta mediante el parámetro q *
Poderes y Habilidades /v1/paa/{id} - Retorna un elemento que concida con el id que se solicitó

Para los elementos marcados con un asterisco se requieren más datos para una respuesta adecuada. Vea la sección Cabeceras HTTP en las peticiones.

Cabeceras HTTP en las peticiones

Siguiendo con las especificaciones REST, en algunas consultas se deben añadir algunas cabeceras HTTP para que se pueda retornar un completo o mejor conjunto de resultaros. La siguiente es una lista de endpoint's que necesitan ciertas cabeceras:

NIVEL ENDPOINT's CABECERAS VALORES
Sección /v1/sections/ Accept-Language Ver lista de valores disponibles
Sección /v1/sections/{acronym} Accept-Language Ver lista de valores disponibles
Sub-sección /v1/subsections/ Accept-Language Ver lista de valores disponibles
Sub-sección /v1/subsections?q Accept-Language * Ver lista de valores disponibles

Para los elementos marcados con un astericos, las cabeceras son opcionales.
